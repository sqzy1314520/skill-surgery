# Contributing to skill-surgery

Thanks for checking this out. This is a meta-skill — it operates on other skills. Here's how to help.

## Ways to contribute

### Report a skill that needs surgery

Open an issue with:
- Which Hermes skill has a recurring error pattern
- What the error looks like
- Whether you've tried light/standard/full encapsulation

### Add an encapsulation template

If your skill has a unique lifecycle pattern (e.g., database transactions, network calls, multi-step user confirmation), open a PR that documents the pattern in `references/`.

### Share your bootstrap story

We're tracking whether encapsulation actually reduces error frequency (the Triple Gate). If you've encapsulated 2+ skills and tracked error rates for 30+ days, open a discussion — that data is gold.

## PR process

1. Fork the repo
2. Make your change
3. Open a PR with a clear description of what changed and why

## Code of conduct

Be constructive. This is experimental infrastructure — expect rough edges.
